# MEMENTO: Teaching LLMs to Manage Their Own Context

## Bibliographic Information

## Abstract

Reasoning models think in long, unstructured streams with no mechanism for compressing or organizing their own intermediate state. The paper introduces **MEMENTO**: a method that teaches models to segment reasoning into blocks, compress each block into a *memento* (a dense state summary), and reason forward by attending only to mementos. This reduces context, KV cache size, and compute.

To train MEMENTO models, the authors release **OPEN MEMENTOS**, a public dataset of 228K reasoning traces derived from OpenThoughts-v3, segmented and annotated with intermediate summaries. A two-stage SFT recipe on OPEN MEMENTOS is shown to be effective across model families (Qwen3, Phi-4, Olmo 3) and scales (8B–32B parameters). Trained models maintain strong accuracy on math, science, and coding benchmarks while achieving ~2.5× peak KV cache reduction. The authors extend vLLM to support their inference method, achieving up to 2× throughput improvement while enabling RL fine-tuning. A key finding is a **dual information stream**: information from each reasoning block is carried both by the memento text and by the corresponding KV states, which retain implicit information from the original block. Removing this channel drops accuracy by 15 pp on AIME24.

---

## Problem Statement

Modern LLMs using chain-of-thought (CoT) reasoning generate thousands of tokens before arriving at an answer. This creates two major problems:

1. **No built-in context organization:** A 32K-token CoT is a flat, unstructured stream. The model has no mechanism to mark intermediate results as important or compress a long derivation into a compact conclusion.
2. **Uniform attention cost:** Every past token sits in the attention window at equal cost. The model has no learned way to drop or compress past reasoning.

As reasoning models scale to longer traces, KV cache memory usage and inference compute grow proportionally, bottlenecking both throughput and deployability.

---

## Key Contributions

1. **OPEN MEMENTOS:** A 228K-trace public dataset of segmented, summarized reasoning chains, with a full annotation pipeline and code. Sourced from OpenThoughts-v3 (QwQ-32B generated traces); 54% math, 19% code, 27% science.

2. **Demonstration at scale:** Training across three model families (Qwen3 8B/32B, Phi-4-reasoning 14B, Olmo-3-7B-Think) showing models internalize summarization as a learned capability while preserving reasoning accuracy at 2–3× peak KV cache reduction.

3. **Native block masking in vLLM:** A custom vLLM fork supporting in-place KV cache masking within a single generation call—a key infrastructure requirement for both inference and RL training with MEMENTO. This enables RL fine-tuning with block masking, allowing closure of the accuracy gap.

4. **The dual information stream:** Identification and experimental verification that memento KV states encode information from masked blocks—a mechanism absent in restart-based approaches. Removing this implicit channel drops accuracy by 15 pp on AIME'24.

## Architecture and Method

### Core Idea

MEMENTO trains models to:
1. **Segment** their chain-of-thought into semantically coherent blocks (T1, T2, ..., Tn).
2. After each block, **generate a memento** (Mi): a terse, dense summary preserving all logically relevant information (intermediate values, formulas, strategies, rejected approaches) in ~15–25% of the original block's tokens.
3. **Mask** the preceding thinking block from all subsequent attention once the memento is produced. The model then attends only to past mementos and the current block.

This is implemented with four special tokens: `<|block start|>`, `<|block end|>`, `<|summary start|>`, `<|summary end|>`.

Crucially, masking happens **in-place** (via a custom vLLM engine) rather than by restarting generation. The KV cache entries of each memento are computed while the full block is still in context, and retained after the block is masked. This creates a **dual information stream**: the explicit memento text plus an implicit representational channel through the cached KV states.

### Data Generation Pipeline (OPEN MEMENTOS)

The pipeline takes raw CoT traces and produces structured traces annotated with mementos, in four stages:

**Stage 0 — Seed Selection:** 228K reasoning traces from OpenThoughts-v3 (generated by QwQ-32B) are sourced. Traces from a strong reasoner transfer across model families.

**Stage 1 — Sentence Splitting:** Traces are partitioned into atomic "sentences" (complete, modular thoughts). Code blocks and multi-line math are protected as atomic units. Logical fragments are merged (sentences ending with colons, continuation words like "Therefore"/"Thus", short fragments < 5 tokens). This reduces candidate boundaries by ~2× vs. naive splitting (~397 → 187 per trace).

**Stage 2 — Boundary Scoring:** An LLM judge (GPT-5.x) scores each inter-sentence boundary from 0 (mid-thought) to 3 (major semantic transition/chapter boundary). Batched scoring with a window of consecutive sentences. Critical rules prevent mid-calculation splits.

**Stage 3 — Segmentation:** Given boundary scores, the trace is partitioned into K contiguous blocks by maximizing:

```
(1/K) * sum(scores at boundaries) - lambda * sigma(block_lengths) / mu(block_lengths)
```

subject to each block containing at least 200 tokens, with lambda = 0.5. The first term rewards cutting at strong semantic boundaries; the second penalizes uneven block sizes (scale-invariant via coefficient of variation). Without the balance term, the optimizer produces one very long block and several tiny ones.

**Stage 4 — Iterative Memento Generation:**
- **Compressor (GPT-5.x):** Receives all blocks and produces one terse memento per block using semicolon-separated clauses, "name: value" pairs, compact math. Targets ~10% of block tokens (≤20%).
- **Judge (GPT-5.x):** Evaluates each memento on a 0–10 scale across six dimensions: (1) formulas extracted verbatim (0–3), (2) numerical values preserved (0–2), (3) methods named (0–2), (4) validation included (0–1), (5) no hallucinations (0–1), (6) result-first structure (0–1). Threshold τ = 8/10.
- **Refinement:** Up to T=2 iterations (Compressor → Judge → Compressor → Judge). Single-pass achieves only 28% pass rate; two iterations reach 92%.

### Dataset Statistics

- 228K samples: 54% math (123K), 19% code (44K), 27% science (61K)
- Blocks per sample: median 9 (math/code), 7 (science)
- Block sizes: median 3.8K chars (math), 2.3K chars (science)
- Summary sizes: stable at 509–603 chars across all domains
- Compression ratios: 0.16 (math), 0.18 (code), 0.23 (science) — ~4–6× block-level compression
- Trace-level: average block ~1,150 tokens → memento ~194 tokens (~6× compression)

### Training Procedure: Two-Stage SFT

The training separates format learning from context management (curriculum learning):

**Stage 1 — Full Attention:** Standard causal attention over all tokens. Loss on all tokens (thinking blocks, mementos, special tokens, final answer). Model learns the block-memento format without any context management pressure.

**Stage 2 — Memento Attention:** After each completed memento, the preceding thinking block is masked from all subsequent attention. Teaches the model to produce self-contained mementos carrying all information needed for downstream reasoning.

For models without reasoning capability (e.g., Qwen2.5-7B-Instruct), an additional prior stage of vanilla reasoning SFT (OT → OM/Full → OM/Mem) is required. For already-strong reasoning models (Qwen3-8B/32B), the two-stage SFT suffices.

**Key hyperparameters:** Learning rate 8×10⁻⁵, cosine schedule with 5% warmup, 5 epochs per stage, AdamW (β1=0.9, β2=0.999), no weight decay, bfloat16 precision, batch size 512, 32 B200 GPUs, 31K training samples, 32K sequence length.

**Special token initialization:** Embeddings initialized as the mean of semantically related existing tokens (e.g., `<|block start|>` from "block", "start", "begin", "section", "step") plus small Gaussian noise (σ=0.01).

### Inference Engine: vLLM with Block Masking

MEMENTO requires non-standard, data-dependent sparse attention (which tokens are masked depends on the generated sequence, not a fixed pattern known at compile time). No production inference framework supports this natively. The authors build **native block masking support directly into vLLM's V1 engine**, implemented at the Python level—no custom sparse attention kernels required.

Three components:
1. **Per-request state machine:** Each request carries a `BlockMaskingState` tracking open blocks, completed blocks, and pending compactions.
2. **Physical KV cache compaction:** Masked tokens are physically removed (not just logically masked) from the KV cache. Active entries are copied into contiguous slots; standard FlashAttention/FlashInfer kernels work unmodified.
3. **Scheduler integration:** Orchestrates compaction between forward passes; updates `num_computed_tokens` for the request.

A `keep_last_n_blocks` parameter controls compaction aggressiveness: −1 (vanilla), 0 (compact all), N>0 (keep last N blocks visible).

## Experiments and Results

### Main Accuracy and KV Cache Results (Table 1)

Evaluated on 14 benchmarks: 11 competition math (from MathArena, grouped as "Comp. Math"), MATH-500, GPQA Diamond, and LiveCodeBench v6. All results: pass@1. Comparison: Base model, Control (SFT on original OpenThoughts without memento annotations), MEMENTO SFT, MEMENTO+RL.

**Qwen3-8B:**
- Base AIME'26: 66.8% | MEMENTO: 57.3% (−7.4 pp vs. Control) | MEMENTO+RL: 64.9% (+0.2 pp vs. Control)
- Peak KV: 2.59 GB (Control) → 1.02 GB MEMENTO (0.39×) → 1.45 GB MEMENTO+RL (0.56×)
- GPQA-D: MEMENTO+RL 62.9% exceeds vanilla baseline of 61.4%

**Phi-4-reasoning (14B):**
- Base AIME'26: 71.7% | MEMENTO: 67.6% (−2.2 pp vs. Control)
- Peak KV: 3.04 GB (Control) → 1.17 GB MEMENTO (0.38×)

**Qwen3-32B:**
- Base AIME'26: 75.2% | MEMENTO: 72.6% (−1.5 pp vs. Control)
- Peak KV: 3.83 GB (Control) → 1.67 GB MEMENTO (0.44×)
- KV AUC reduction: up to 3.5× on competition math

**Olmo-3-7B-Think (hybrid sliding-window attention):**
- Accuracy well preserved (Comp. Math drops only 0.2 pp vs. Control)
- KV savings modest: ~0.85–0.93× peak (75% of layers use sliding-window attention capped at 4096 tokens regardless of block masking; only 8 full-attention layers benefit)

**Key patterns:**
- **Scale helps:** Gap shrinks from −6.3 pp at 8B to −3.5 pp at 32B (averaged across five benchmark groups)
- **Easier benchmarks:** MATH-500 gap < 1 pp across all models
- **KV AUC savings:** 2–3.5× on competition math
- **Compression behavior:** Summary sizes stable at ~260–615 chars across model families and benchmarks; block sizes vary widely. Compression strongest on competition math (9–27×), weakest on shorter-block benchmarks (6–9×).

### Throughput Results

Benchmarked on Qwen3-8B, single B200 GPU, AIME24 × 8 repetitions (240 requests, 32K max tokens):
- Vanilla vLLM plateaus as KV cache fills GPU memory at high concurrency
- MEMENTO sustains **4,290 tok/s vs. 2,447 tok/s** (1.75× higher throughput)
- Batch completion: **693s vs. 1,096s** (1.58× faster)

### RL Fine-Tuning (Qwen3-8B)

**Problem:** Post-SFT accuracy gap is a *consistency* problem, not a *capability* problem. Pass@64 coverage gap averages only 2.6 pp; Jaccard similarity between Base and MEMENTO solved sets averages 96.4%. Majority voting (maj@k) shows MEMENTO models match or exceed Base pass@1 with just k=2–3 samples.

**Method:** CISPO (Clipped Importance-Sampled Policy Optimization, a GRPO variant) on the Qwen3-8B MEMENTO SFT checkpoint. Uses DolciMath training set (29,670 prompts), rule-based math rewards, KL penalty β=0.001, block length capping at 7K tokens per block during rollouts.

**Results after 350 steps:**
- AIME'26: 57.3% → 64.9% (+7.6 pp)
- Comp. Math: 45.1% → 49.4%
- GPQA-D: 55.8% → 62.9% (above vanilla baseline of 61.4%)
- Peak KV rises from 1.08 to 1.48 GB (still 45% below vanilla's 2.71 GB)

### Dual Information Stream: KV Ablation

**Experiment:** Compare two inference modes on the same Qwen3-8B checkpoint:
- **Normal:** Mementos' KV states computed while full block is in context; block then masked but KV states retained.
- **Restart:** Mementos' text generated identically, but KV states recomputed from scratch (prefill over prompt + mementos only, without block content).

**Result:** Memento attention (normal) = 66.1% pass@1; Memento attention + restart = 50.8% pass@1. Delta: **−15.3 pp**. This confirms memento KV states carry significant information from masked blocks. MEMENTO's mementos function as compressed pointers into cached reasoning state, not standalone text replacements.

### Probing the Implicit KV Channel

**Experiment design:** Inject a known random 5-digit passcode into a target block T2 in a real AIME'25 trace. Run forward pass with block masking. Extract KV states from memento token positions and train an MLP probe to predict the 5 digits. Three conditions:
- **Direct:** Probe M2 (can attend to T2) — upper bound
- **Masked:** Probe M3 (cannot attend to T2, already evicted) — tests information propagation
- **Causal control:** Probe M1 (precedes T2) — should be chance-level (10%)

**Results:**
- Direct: 65.2% (Qwen3-8B), 71.2% (Qwen3-32B) — KV states encode far more than memento tokens themselves
- Masked: 26.7% (Qwen3-8B), 23.0% (Qwen3-32B) — well above 10% chance; information propagates through the memento chain
- Causal control: 10.7% / 10.8% — exactly chance-level

**Layer depth:** Signal concentrates in deeper layers. Early layers (4th) show near-chance masked accuracy (~10.8%); last layers show 26.5% (8B) and 22.4% (32B).

**Toy transformer validation:** A 4-layer model (810K parameters, synthetic data) exhibits the same leakage patterns (24.9% masked accuracy vs. 10% chance), with signal decaying over distance but persisting up to 7 hops from the target block. Leakage stays constant across training checkpoints as task accuracy improves, confirming the channel is **architectural, not learned**.

## Limitations

- **Accuracy gap exists at smaller scales:** At 8B parameters, MEMENTO SFT trails the vanilla baseline by ~6–7 pp on hard competition math; this gap is significantly reduced by RL but not fully eliminated for some benchmarks.
- **Hybrid architectures benefit less:** Models with sliding-window attention (e.g., Olmo-3-7B) see only 0.85–0.93× peak KV reduction because most layers already cap their KV cache regardless of block masking.
- **Block length capping trade-off:** During RL, the model can learn to generate fewer, longer blocks to undermine KV savings; a 7K-token per-block cap is required, imposing a constraint on the RL optimization.
- **Higher total KV usage in some cases:** When MEMENTO induces excessive generation (e.g., 3× more tokens on some problems), the KV area-under-curve metric can be worse despite a lower peak.
- **Engineering complexity:** Requires custom vLLM fork; incompatible with vLLM's prefix caching; requires model-specific architecture forks for training.
- **Scales RL only to Qwen3-8B:** RL experiments reported only for the 8B model; scaling RL to 32B remains future work.

## Conclusion

MEMENTO teaches language models to manage their own context by segmenting reasoning into blocks, compressing each into a dense memento, and masking completed blocks via sparse attention. Across three model families (Qwen3, Phi-4-reasoning, Olmo-3-7B-Think):

- **2–3× peak KV cache reduction**, up to 3.5× KV AUC reduction
- **1.75× higher serving throughput** on a single B200 GPU
- **Strong reasoning accuracy preserved:** Qwen3-32B loses just 2.6 pp on AIME'26 and 3.5 pp averaged across five benchmark groups; gap shrinks with scale (6.3 pp at 8B → 3.5 pp at 32B)
- **RL closes remaining gap:** CISPO RL on Qwen3-8B recovers the accuracy gap while retaining KV savings

The dual information stream finding is a key scientific contribution: mementos carry information from masked blocks through both the explicit summary text and the implicit KV representations computed while the block was still visible. Removing this implicit channel degrades accuracy by 15 pp, distinguishing MEMENTO from methods that simply discard context after summarization.

**Future directions identified:**
- Scaling the RL recipe to larger models (32B+)
- Applying MEMENTO to long-horizon agent tasks where agent steps form natural blocks

**Released artifacts:** OPEN MEMENTOS (228K annotated reasoning traces) and a vLLM fork with native block masking support.

## Notable References and Related Work

### Most Closely Related Work

- **InftyThink** (Yan et al., 2025): SFT-only approach training models to compress reasoning chunks into summaries; operates at the text level (discards KV cache after summarization — "restart-based").
- **InftyThink+** (Yan et al., 2026): Adds RL on top of InftyThink; also restart-based.
- **Accordion-Thinking** (Yang et al., 2026): Like InftyThink+ but implements mid-generation context rebuilding within a single inference pass; still text-level, discards original KV cache.
- **PENCIL** (Yang et al., 2025): Trains models from scratch on synthetic tasks to erase intermediate reasoning via reduction rules; small models solve hard 3-SAT with bounded context.
- **Gist tokens** (Zhang et al., 2025a; Monea et al., 2025): Compresses reasoning chunks into learned special tokens whose KV states encode compressed representations; loses interpretability. MEMENTO's summaries are natural-language text.

### Context Management Systems

- **MemAgent** (Yu et al., 2025): RL to compress long input documents into fixed-size memory.
- **MEM1** (Zhou et al., 2025): Compact internal state across tool calls for multi-turn agents.
- **ReSum** (Wu et al., 2025) and **A-MEM** (Xu et al., 2025): External memory/summarization infrastructure.

### KV Cache Compression (Orthogonal / Composable)

- **ThinKV** (Ramachandran et al., 2025), **R-KV** (Cai et al., 2025), **Reasoning Path Compression** (Song et al., 2025): Inference-time KV pruning/quantization based on attention patterns.
- **TokenSkip** (Xia et al., 2025), **Step Entropy Compression** (Li et al., 2025): Train models to skip low-importance tokens.
- **C3OT** (Kang et al., 2025), **TokenSqueeze** (Zhang et al., 2025b): Train on compressed reasoning traces.
- **ThinkPrune** (Hou et al., 2025), **GRFO** (Shrivastava et al., 2025): RL to shorten traces.
- **Sliding-window attention** (Longformer, Beltagy et al., 2020; OLMo 3, Team Olmo et al., 2025): Architectural KV bound.

### Infrastructure

- **vLLM / PagedAttention** (Kwon et al., 2023): Production inference framework extended by this work.
- **SGLang** (Zheng et al., 2024): Alternative inference framework.
- **GRPO** (Shao et al., 2024) and **CISPO** (MiniMax et al., 2025): RL algorithms used for fine-tuning.
- **OpenThoughts** (Guha et al., 2025): Source dataset of 228K CoT traces.
- **MathArena** (Balunovic et al., 2025): Source of competition math benchmarks (AIME, HMMT, BrUMO, SMT, CMIMC) designed to minimize contamination.
